連携に必要なapiが提供されていない
(A) データエクスポート方式

ユーザーが『ME』からCSV/明細エクスポートを行い

それをサブスク管理アプリにアップロードして解析・統合する

※ この方式は自動連携ではありませんが、ユーザー自身が手動でデータを持ってくる形で連携できます。

(B) 金融 API 専用サービスを中間に使う

銀行・カード明細を取得する際によく使われるサードパーティ API（例：Plaid や Salt Edge 等）は、家計簿ではなく明細レベルで金融データを取得し、それを元に自前アプリ内で家計簿側データを構築するという形が現実的です。

→ つまり

サブスク管理アプリが Plaid 経由で明細を取得

明細からサブスク支出を自動判定

自前 UI で家計簿的な一覧にする

というアプローチが可能です。
